# TelecomSystem

Проект содержит графическое приложение-сервер и консольное приложение-клиент.
Один или несколько клиентов подключаются к серверу, генерируют данные и отправляют их серверу.
Сервер управляет клиентами, получает от них данные и отображает.

Папка `build` содержит папки с готовыми исполняемыми файлами:
`build/Server/Server.exe` для сервера, `build/Client/Client.exe` для клиента.
Исполняемые файлы собраны для Windows 11, можно запустить в Windows 10 новых версий.

## Инструкция

1. Запустить сервер (`build/Server/Server.exe`).
2. Запустить один или несколько клиентов (`build/Client/Client.exe`).

Для каждого клиента сервер отобразит в окне событий сообщения о подключении, 
об отправке подтверждения подключения и конфигурации лимитов клиенту.
Подключенные клиенты отобразятся в таблице **"Информация о клиентах"**
Клиент отобразит у себя сообщение об успешном подключении к серверу 
и получении подтверждения подключения и конфигурации лимитов.

Можно запустить сперва клиентов, потом сервер.
Клиенты будут пытаться подключиться к серверу каждые 5 секунд.

3. Нажать на кнопку **"Начать передачу"** в верхней части окна сервера 
   для начала генерации данных клиентами и отправки их серверу.

Клиент отобразит сообщение о получении запроса на генерацию, 
затем о начале генерации данных, после чего начнёт отправку.
Сервер начнет отображать эти данные в таблице **"Данные клиентов"**
Процесс можно остановить повторным нажатием на ту же кнопку на стороне сервера
(ее текст изменится на "Остановить передачу").

4. Нажатие на кнопку **"Конфигурация лимитов"** откроет окно, 
   где можно задать лимиты конкретных значений для сообщений типов NetworkMetrics и DeviceStatus.
   Нажатие на CheckBox слева от названия поля активирует валидацию по этому полю.

Если во время передачи данных клиент сгенерирует сообщение, 
у которого указанные значения превысят заданный лимит, клиент сгенерирует 
лог типа WARNING с указанием, какие значения были превышены.
Конфигурацию можно применять во время активной передачи.

После подключения клиенту передаётся текущая конфигурация лимитов, 
а также команда о начале передачи данных, если передача активна на стороне сервера.
Таким образом, все клиенты всегда остаются синхронизированы с сервером.

Все сетевые ошибки выводятся в консоли клиента и в окне **"События"** сервера.

## Сборка

```sh
mkdir build-custom
cd build-custom
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . --config Release
```

Для сборки через Qt Creator:
1. Нажмите "Open Project..."
2. Откройте CMakeLists.txt в корневом каталоге проекта
3. В качестве папки с билдом укажите любое название кроме `build` (в этой папке находятся готовые сборки)
4. Выполните сборку в режиме Debug или Release
